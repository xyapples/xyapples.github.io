# 1.2.3 结果总结
  • 1. 带 / 意味着 Nginx 代理会修改用户请求的 URL，将 location 匹配的 URL 进行删除。
  • 2. 不带 / 意味着 Nginx 代理不会修改用户请求的 URL，而是直接代理到后端应用服务器。
